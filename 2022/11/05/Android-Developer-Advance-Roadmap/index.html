Fragment

Fragment 的使用很简单，但是要管理好它的交互还是比较复杂的。下面整理的是比较关键的几个方面。

Fragment manager

FragmentManager 用于 add() / replace() / remove() Fragment。每一个 FragmentActivity 都有一个 FragmentManager，可以通过 supportFragmentManager 获得它的引用. 同样的，每个 Fragment 也有自己的 FragmentManager，可以通过 childFragmentManager 获得引用，并且它比 Activity 多了一个 parentFragmentManager。参考下图可以清晰理解它们的关系：

Fragment transactions

Transaction 是 Fragment manager 的操作单元，一个 Transaction 可以包含一个或者多个 fragment 的 add/replace 等操作。
